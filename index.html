1 WordPress learning/development experiments

1.1 Dragon’s Brew Coffee Corner blog: wordpress.com version

I built this version of the blog just to get started with WordPress, and to see what features the non-installed, auto-hosted WordPress blogs offered. Obviously, the wordpress.com site offers much less flexibility than the wordpress.org installed apps, which are fully customizable, offer unlimited themes and plugins, etc.

1.2 Dragon’s Brew Coffee Corner blog: wordpress.org version

1.2.1 WordPress Dashboard/Block Editor

1.2.1.1 Document Overview tab

When editing the Dragon’s Brew Coffee Corner blog, I ran into the issue of needing to change the format of what my posts displayed; through the document overview tab in the block editor, I was able to change the post template to display the information desired.

1.2.1.2 Custom fields with the ACF plugin

I was curious about the possibility of displaying dynamic data in a blog post, and came across the ACF plugin. Using the ACF plugin, the “field groups” functionality enables one to create the new field, followed by specifying the desired data inside the individual post. We can then decide where we want this field displayed in the block editor of the template across all posts.

1.2.1.3 Appearance Editor

In the Appearance Editor we can make all sorts of different changes to the different pages of our website, the templates we are using, or adding additional CSS if desired.

1.2.2 Hosting (on dreamhost.com)

After doing some research, I decided on DreamHost for hosting my Dragon’s Brew Coffee Corner blog. DreamHost is quite nice as it installs WordPress for you, gives quick and easy access to the WordPress dashboard through their own dashboard, has metrics for showing the traffic to your website, and even gives you a free SSL certificate.

1.3 Fetching WordPress posts from another website

I thought it would be interesting to take my existing Dragon’s Brew Cafe MERN stack application and create a page that displays the latest blog post from the Dragon’s Brew Coffee Corner (wordpress.org version) blog site. In my React front end, I fetched the latest post from the WordPress API endpoint, and then stored the response locally. I then took a peek at what JSON was returned to see what fields I needed to access to display them properly.

2 My opinion of WordPress

2.1 Overall impression

WordPress seems fine. No tool is perfect, and the fact that WordPress started its life as a blog and evolved into a CMS of sorts is somewhat evident in places, but it’s certainly workable enough. Of course, my opinion might change as I delve deeper into its customizations, as I’ve only scratched the surface there.

2.2 The good

There are plenty of things to like about WordPress. One can certainly get started quickly using its built in themes and plugins, and then evolve the site in more desirable design directions by customizing the desirable pieces. There’s a built in block editor for structural changes, a form of version control through the revisions mechanism, and it’s nice to be able to view the code in the code editor and the blocks in the visual editor.

2.3 The bad

As I said above, my opinion might certainly change as I work with it more, but at least currently, most things in WordPress seem fairly workable to me. Its editing UI could certainly use some work and polish in a number of places, and the number of features in the dashboard is a bit overwhelming and might benefit from some reorganization, but I don’t have any major complaints.
